Users Routes

Method URL Body Description
GET /users/getUserInfo None Get user info - by token.
GET /users None Get all users - by token admin.
POST /users 
{
    "name": "Michal",
    "email": "Michal@gmail.com",
    "password": "pass1234",
    "role": "admin"
}
 Create a new user - register
POST /users/login 
{
    "email": "Michal@gmail.com",
    "password": "pass1234",
}
 Login - if all fields correct, the token will save on cookies.

Toys Routes

Method URL Body Description
GET /domain/toys None Get all toys - will present 10 results per page.
GET /domain/toys/single/{id} None Get toy by id
GET /domain/toys/category/{catname} None Get toy by category name - will present 10 results per page.
GET /domain/toys/prices?min={min}&max={max} None Get toy by min/max prices - will present 10 results per page.
GET /domain/toys/search?s={name/info} None Search toys by name or info - will present 10 results per page.
POST /domain/toys 
{
    "name": "Harry",
    "info": "Harry Potter",
    "category": "Stories",
    "image_url": "harryPotter.png",
    "price": 100
}
Add a new toy to the user that login now.
PATCH /domain/toys/{editId} 
{
    "image_url": "harry_potter.png",
    "price": 25
}
 Edit toy after login.
DELETE /domain/toys/{delId} None Delete toy after login.
